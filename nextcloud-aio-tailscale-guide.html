Nextcloud AIO on Docker
with Tailscale Serve

A simple, no-port-forwarding guide to self-host Nextcloud All-in-One using Tailscale as a reverse proxy — no domain purchase required, no VPS needed.

📎 Based on official guide: nextcloud/all-in-one #6817
⚡ You'll need these first
  • Docker Desktop installed
  • Tailscale account (free)
  • Tailscale app on your PC
  • Windows, macOS, or Linux
  • ~10 GB free disk space
  • Terminal / PowerShell
01
Configure your Tailscale account
login.tailscale.com → DNS tab
›
Before touching Docker, you need two settings turned on in your Tailscale admin panel. Both are free. This is what gives you a proper HTTPS domain like mypc.tail1234.ts.net.
🌐
Step 1a — Enable MagicDNS
Go to login.tailscale.com → DNS tab. Find "MagicDNS" and turn it on. This gives each of your devices a stable hostname like mypc.tail1234.ts.net.
🔐
Step 1b — Enable HTTPS Certificates
On the same DNS tab, find "HTTPS Certificates" and enable it. This lets Tailscale auto-issue a valid TLS cert for your machine — so you get a real padlock 🔒 in the browser, no manual cert setup needed.
📋
Step 1c — Note your machine hostname
In the terminal on your server, run this to see your Tailscale hostname. You'll need it later.
tailscale status
Look for your machine's entry — the domain looks like mypc.tail1234.ts.net. Write it down.
ℹ️  Make sure Tailscale is installed and logged in on your server machine before continuing.
02
Run Tailscale Serve — BEFORE installing Nextcloud
⚠️ Critical order: serve must run first
›
This is the most important step most guides get wrong. You must run tailscale serve before you enter your domain in the Nextcloud wizard — otherwise Nextcloud's domain validation will fail because it can't reach port 443.
▶️
Run this command in your terminal
This tells Tailscale: "take HTTPS traffic coming to my machine and forward it to port 11000 (where Nextcloud's Apache will run)."
tailscale serve --bg http://127.0.0.1:11000
# Run in PowerShell as Administrator tailscale serve --bg http://localhost:11000
✅
Verify it's working
Confirm Tailscale is proxying correctly:
tailscale serve status
You should see something like: https://mypc.tail1234.ts.net → http://127.0.0.1:11000
⚠️  The --bg flag runs it in the background. On Linux/systemd servers, see the optional systemd service setup in the Troubleshooting section to make this survive reboots.
03
Create your docker-compose.yaml
based on the official Nextcloud AIO compose file
›
Create a folder for your Nextcloud setup, then create a compose.yaml file inside it. The key additions vs the default are the DNS resolvers and the two Apache environment variables.
📁
Create your project folder
# Pick any directory you like mkdir ~/nextcloud && cd ~/nextcloud
📄
Create compose.yaml with these contents
Copy the official AIO compose file, then make these specific changes (shown with + lines):
services: nextcloud-aio-mastercontainer: image: ghcr.io/nextcloud-releases/all-in-one:latest init: true restart: always container_name: nextcloud-aio-mastercontainer volumes: - nextcloud_aio_mastercontainer:/mnt/docker-aio-config - /var/run/docker.sock:/var/run/docker.sock:ro ports: - 8080:8080 # AIO management interface # Comment out or remove these two lines: # - 80:80 # - 8443:8443 # ✅ Add these DNS resolvers (fixes domain validation) dns: - 100.100.100.100 # Tailscale Magic DNS - 1.1.1.1 # Fallback DNS environment: # ✅ Uncomment and set these two lines: APACHE_PORT: 11000 APACHE_IP_BINDING: 127.0.0.1 volumes: nextcloud_aio_mastercontainer: name: nextcloud_aio_mastercontainer
ℹ️  The full official compose.yaml is at: github.com/nextcloud/all-in-one/blob/main/compose.yaml — start from that and make the changes shown above.
⚠️  Windows users: change /var/run/docker.sock:/var/run/docker.sock:ro to //var/run/docker.sock:/var/run/docker.sock:ro
04
Start the Nextcloud AIO container
docker compose up -d
›
With your compose.yaml saved, start the master container. This pulls the image and starts the AIO management interface.
🐳
Start the container
From inside your nextcloud folder:
docker compose up -d
👀
Check it's running
docker ps
You should see nextcloud-aio-mastercontainer with status (healthy).
05
Run the Nextcloud setup wizard
open https://localhost:8080 from your browser
›
The AIO management interface walks you through the full Nextcloud setup. You'll enter your Tailscale domain here — the one you noted in Step 1.
🌐
Open the AIO interface
In your browser, navigate to:
https://localhost:8080
You'll get a self-signed cert warning. Click Advanced → Proceed (or equivalent in your browser). This is expected.
🔑
Copy the one-time passphrase
The page shows a one-time passphrase. Copy it and store it somewhere safe — you'll need it to log in and you only see it once.
🌍
Enter your Tailscale domain as the domain
When asked for a domain, enter your Tailscale hostname from Step 1. Format: just the bare domain, no https://, no trailing slash.
# Example — replace with your actual hostname mypc.tail1234.ts.net
Click Submit. Because you ran tailscale serve in Step 2, the domain check should pass ✅
📦
Select containers and start Nextcloud
Choose which optional containers you want (e.g. Collabora for Office, Talk for video calls), set your timezone and backup location, then click "Download and start containers". This takes a few minutes.
⚠️  Want a custom Tailscale domain name? Change it before this step by going to DNS tab → "Rename Tailnet". Changing it after will break things and require a full reinstall.
06
Access your Nextcloud!
you're done 🎉
›
Once all containers turn green in the AIO panel, your private Nextcloud is ready. Access it from any device running Tailscale.
🚀
Open Nextcloud in your browser
From any device on your Tailscale network:
# Replace with your actual domain https://mypc.tail1234.ts.net
You should see the Nextcloud login screen with a valid 🔒 padlock. Create your admin account.
📱
Connect mobile & desktop apps
Install the official Nextcloud app on your phone or other computers. When asked for server URL, enter your Tailscale hostname. Make sure Tailscale is running on that device too.
✅  Your Nextcloud is private — only accessible from devices on your Tailscale network. No ports open to the internet. No VPN server to manage.

🔧 Troubleshooting common issues

›
Domain validation fails
Make sure you ran tailscale serve --bg http://127.0.0.1:11000 before entering the domain in the wizard. Also confirm the dns: section is in your compose.yaml.
Port 443 not reachable error
Comment out the 80:80 and 8443:8443 lines in your compose.yaml ports section. Another service may be using those ports. Also try setting APACHE_IP_BINDING: 0.0.0.0 as a last resort.
DNS config not set error
Add the dns: block to your compose.yaml (see Step 3) with 100.100.100.100 as the first entry. Make sure MagicDNS is enabled in your Tailscale admin.
Still can't pass domain check
Last resort: add SKIP_DOMAIN_VALIDATION: true to your environment section. Only use this after confirming everything else is correctly configured (tailscale serve running, DNS set up).
Collabora / Office not working
Comment out ports 80:80 and 8443:8443 from the compose ports section. These conflict with how Collabora's CODE container works.
tailscale serve doesn't survive reboots
Create a systemd service. Save this to /etc/systemd/system/tailscale-nextcloud.service and run systemctl enable --now tailscale-nextcloud:

[Unit]
Description=Tailscale serve for Nextcloud
After=network.target

[Service]
ExecStart=tailscale serve http://127.0.0.1:11000
Type=simple

[Install]
WantedBy=multi-user.target
General debugging commands
docker logs nextcloud-aio-mastercontainer — check container logs
tailscale serve status — verify serve is active
tailscale status — check tailnet connectivity
curl -vk https://mypc.tail1234.ts.net — test HTTPS from the server
